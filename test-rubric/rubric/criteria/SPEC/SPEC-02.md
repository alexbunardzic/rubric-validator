# Criterion

The specification must address material changes to the system architecture.

## Why it matters

Material changes require careful documentation to ensure system integrity.

## How it's assessed

Verify that all [material change](glossary.md#material-change) instances are documented. This criterion depends on [SPEC-01](criteria/SPEC/SPEC-01.md).

## Meets

The specification explicitly documents all material changes with rationale.

## Does not meet

Material changes are not documented or are mentioned without sufficient detail.

## Scope note

See also [VER-01](criteria/VER/VER-01.md) for verification approaches.

## Mappings

Relates to configuration management standards.
