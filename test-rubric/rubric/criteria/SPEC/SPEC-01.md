# Criterion

The specification must be complete and unambiguous.

## Why it matters

A clear specification ensures that developers can implement the feature correctly.

## How it's assessed

Review the specification document for completeness and clarity.

## Meets

The specification covers all use cases and edge cases with concrete examples.

## Does not meet

The specification is incomplete or contains ambiguous language that leaves implementation details unclear.

## Scope note

This criterion applies to all new features. See [SPEC-02](criteria/SPEC/SPEC-02.md) for more on verification.

## Mappings

Maps to ISO/IEC/IEEE 42010 architecture documentation standards.
