# Criterion

Verification procedures must test all documented requirements.

## Why it matters

Comprehensive verification ensures the implementation meets the specification.

## How it's assessed

Review test cases and coverage metrics against the specification.

## Meets

Tests cover all specified behaviors with passing results documented.

## Does not meet

Test coverage is incomplete or verification results are not documented.

## Scope note

This builds on specification completeness (SPEC-01).

## Mappings

Aligns with verification and validation standards.
